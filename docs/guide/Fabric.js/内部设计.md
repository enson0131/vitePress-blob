# 内部设计

## 使用场景上看

从使用场景上看，Fabric.js 通过对象的形式来创建图形 （源码上也充满了面向对象设计的身影）

对于使用者而言，不需要关心底层 Canvas 是如何实现的，只需要关心如何创建这些图像对象即可。

## Canvas 开发原理

Canvas 开发的本质其实很简单，可以想象下这种少儿画板：

![初始状态图](./../../public/assets/fabric/1.png)

Canvas 的绘制其实就是在不断在画布上绘制图形，然后擦出的过程。
